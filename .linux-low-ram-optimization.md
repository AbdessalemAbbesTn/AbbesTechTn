عند تشغيل أنظمة " لينكس " على أجهزة ذات مواصفات قديمة مثل(ذاكرة عشوائية 2 جيجابايت)، نواجه تحدياً كبيراً في كيفية إدارة الذاكرة، المشكلة الأساسية هي "إجهاد القرص

" (Disk Thrashing)؛ 

فعندما تمتلئ الذاكرة المؤقة (الرام)، يبدأ النظام بنقل البيانات إلى القرص الصلب

 (Swap)،
 
 و لأن القرص أبطأ بكثير من الرام، يختنق النظام و تظهر أخطاء الإدخال و الإخراج
 
 (I/O Errors)، 

و قد يتوقف النظام عن الإستجابة تماماً.
ما هو معامل

Swappiness
؟

فكر في هذا المعامل كمؤشر يخبر النظام: "متى يجب أن تبدأ في إستخدام القرص الصلب كبديل للرام؟".
 * القيمة الإفتراضية (60) تجعل النظام يلجأ للقرص مبكراً.
 * تخفيض القيمة إلى (10) يجعل النظام يتمسك بالرام السريعة لأقصى درجة ممكنة، و لا يلجأ للقرص إلا عند الضرورة القصوى، مما يحسن الأداء بشكل ملحوظ.
خطوات العمل
 1.فحص الذاكرة: نستخدم الأمر التالي لمعرفة حجم الذاكرة المستخدمة حالياً :
   
   free -h
   
 2.التطبيق المؤقت: لتجربة الأداء فوراً دون إعادة تشغيل:
 
   sudo sysctl vm.swappiness=10
   
   
 3.الحفظ الدائم: لكي لا نفقد الإعدادات عند إيقاف التشغيل، نكتب القيمة في ملف إعدادات النظام:
 
   echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf
   
 4.التأكد: للتثبت من أن النظام إعتمد الرقم الجديد:
 
   cat /proc/sys/vm/swappiness

[ENGLISH SECTION]
Running modern Linux distributions on legacy hardware (e.g., 2GB RAM) requires strategic memory management. The main issue is "Disk Thrashing," where the system excessively swaps data between RAM and the disk. Since disks are slower than RAM, this causes latency and "Input/output errors," sometimes forcing the system into a "Read-only" mode to prevent data corruption.
Understanding Swappiness
The swappiness parameter controls how often the kernel moves processes out of physical RAM and onto the swap partition.
 * The default value (60) is too aggressive for low-RAM systems.
 * By lowering this to (10), we force the system to utilize the faster physical RAM for a longer duration, reducing disk strain and keeping the UI responsive.
Implementation Steps
 1.Check Memory Status:
   free -h
 2.Apply Immediately: To test the setting without a reboot:
   sudo sysctl vm.swappiness=10
 3.Make it Permanent: Add the configuration to the system control file:
   echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf
 4.Verification: Confirm the kernel is using the new value:
   cat /proc/sys/vm/swappiness
